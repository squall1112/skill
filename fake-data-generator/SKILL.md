# 机器人假数据配置自动生成器

根据国家/公会/玩家数量，自动生成 `t_guild` + `t_user` 的 SQL 配置文件。

---

## 表结构

### t_guild
```sql
INSERT INTO `t_guild` (`id`, `name`, `country_code`, `fake`, `member_capacity`, `member_count`) VALUES (?, '?', '?', true, ?, ?);
```
####字段说明
| 字段名 | 数据类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| **id** | 整数（主键） | 公会唯一标识，从1开始递增 | 1, 2, 3... |
| **name** | 字符串 | 公会名称，可替换 | 'BubbleBusters' |
| **country_code** | 字符串（2位） | 国家代码，可替换 | 'US', 'CN' |
| **fake** | 布尔值 | 标记是否为假数据，全部为 true | TRUE |
| **member_capacity** | 整数 | 公会成员容量 | 50 |
| **member_count** | 整数 | 公会当前成员数量 | 50 |

### t_user
```sql
INSERT INTO `t_user` (`id`, `nickname`, `country_code`, `fake`) VALUES (?, '?', '?', true);
```

####字段说明

| 字段名 | 数据类型 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| **id** | 整数（主键） | 用户唯一标识，从 10000 开始递增 | 10000, 10001... |
| **nickname** | 字符串 | 用户昵称可替换，需要修改的主要字段，包含两种格式 | 'Marshmallow', 'Player_345214' |
| **country_code** | 字符串（2位） | 国家代码可替换，标识用户所属国家 | 'US', 'CN', 'TR' |
| **fake** | 布尔值 | 标记是否为假数据，全部为 true | TRUE |

---

## 支持的国家

AR, AU, BR, CA, CH, CN, DE, ES, FR, IN, IT, JP, KR, MX, NL, NO, RU, SE, TR, US


---

## 命名规则

### t_user 玩家昵称（核心原则：网络中玩家真正会取的名字）

#### 两种格式
**两种格式自定义昵称：默认昵称=2:8比例分配**
**自定义昵称：食物、网络热词、动漫人物、形容词、各种物品、国内外热梗词语**
**默认昵称：Player_随机6位数字，数字唯一不可重复**



#### t_user默认昵称规则

**Player_XXXXXX**,XXXXXX为随机数字

### t_guild 工会昵称

#### 命名特征

**形容词、食物、避免真名、动漫剧情中的组织、电视剧中的组织、电影中的组织、网络热词、网络热梗词**

##生成流程

### Step1：询问用户需求

|参数|提问方式|示例|
|---|---|---|
|国家列表|你想包含几个国家？|全部20国 / AR,US,CN,JP / 自定义 |
|每国公会数|你希望配置中每个国家有几个工会？|3-7范围随机/4-5范围随机/自定义|
|每个工会玩家数|你希望配置中每个工会中有多少个玩家？|30-40范围随机/20-30范围随机/自定义|
|特殊要求|默认名称中的数字ID是否有范围要求？|

### Step 2：生成数据
1. 用固定种子保证可复现（默认 seed=001，或用户指定）
2. 为每个国家生成公会（公会名不重复）
3. 为每个公会生成玩家（20%文化特色词 + 80% Player_格式）
4. 组装 SQL

输出格式：

```sql
-- Fake Data SQL Dump

START TRANSACTION;
-- Guilds
INSERT INTO `t_guild` ...;
-- Users
INSERT INTO `t_user` ...;

COMMIT;
```

## 注意事项

1. 注意点一：注意生成的玩家名字不要出现真实玩家名字
2. 注意点二：注意生成的公会名字避开政治敏感词
3. 注意点三：列表不要以字母排序，需要随机顺序
4. 注意点四：不同国家的玩家和公会名字需要以其对应国家的文字来命名
